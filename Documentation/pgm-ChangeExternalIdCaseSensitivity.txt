= ChangeExternalIdCaseSensitivity

== NAME
ChangeExternalIdCaseSensitivity - Convert `username` and `gerrit`
external IDs to be handled case insensitively

== SYNOPSIS
[verse]
--
_java_ -jar gerrit.war _ChangeExternalIdCaseSensitivity_
  -d <SITE_PATH>
  [--revert]
  [--dryrun]
--

== DESCRIPTION
Convert `username` and `gerrit` external IDs to be handled case
insensitively or case sensitively. This is done by recomputing
the name of the note from the sha1 sum of the all lowercase
external ID key or while preserving the key capitalization
respectively.

The `auth.userNameCaseInsensitive` option has to be set to `false`
when using the tool to convert to case insensitive behavior and will
be set to `true` by the tool. If `--revert` is set,
`auth.userNameCaseInsensitive` option has to be set to `true`.

== OPTIONS

-d::
--site-path::
	Location of the gerrit.config file, and all other per-site
	configuration data, supporting libraries and log files.

--revert::
    Whether case insensitive external IDs should be converted
    back to case sensitive external IDs.

--dryrun::
    Whether to perform the conversion without persisting it.

== CONTEXT
This command can only be run on a server which has direct
connectivity to the managed Git repositories.

== EXAMPLES
To convert the external IDs to be case insensitive:

----
	$ java -jar gerrit.war ChangeExternalIdCaseSensitivity -d site_path
----

To convert the external IDs to be case sensitive again:

----
    $ java -jar gerrit.war ChangeExternalIdCaseSensitivity -d site_path --revert
----
