Gerrit Code Review - /docsearch/ REST API
=========================================

This page describes the documentation search related REST endpoints.
Please also take note of the general information on the
link:rest-api.html[REST API].

Please note that this feature is only usable with documentation built-in.
You'll need to
`buck build :withdocs`
or
`buck build :release`
to test this feature.

[[docsearch-endpoints]]
Documentation Search Endpoints
------------------------------

[[search-documentation]]
Search Documentation
~~~~~~~~~~~~~~~~~~~~
[verse]
'GET /docsearch/'

With `q` parameter, search our documentation index for the query.

To be able to co-exist with our current search for changes, all documentation
search queries must be with a "doc:" operator prefix.
All keywords without the operator will be ignored.

A list of link:#doc-result[DocResult] entities is returned describing the
results.

.Request
----
  GET /docsearch/?q=doc:rest HTTP/1.0
----

.Response
----
  HTTP/1.1 200 OK
  Content-Disposition: attachment
  Content-Type: application/json; charset=UTF-8

DocResult
~~~~~~~~~
The `DocResult` entity contains information about a document.

[options="header",width="50%",cols="1,^2,4"]
|=========================
|Field Name  ||Description
|`title`     ||The title of the document.
|`url`       ||The URL of the document.
|=========================
